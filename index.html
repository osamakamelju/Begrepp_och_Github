Github flow - Vad är det? Hur använder man det? Vilka viktiga delar är inkluderade i begreppet?

Foton hämtat från: Github Redigerat: Okt. 7, 2022

- "Github flow" är förkortas till "Github workflow", det är ett onlineverktyg för att hantera och kontrollera projektet från en person eller flera genom att dela.

- Genom att skapa branch, dela det med teamet, diskutera och granska commits och sist att kommentera.

- Till exempel om det finns ett grupparbete, och en som chef, var och en ska ansvara för en del av arbetet. Så om vi har en webbutikshemsidan skulle vi dela upp arbetet mellan grupparbete så att var och en kommer att arbeta för sin del (en för navigationbar, en för innehållet/body, en för sidfoten/footer, en för de andra funktioner, t.ex. som databas Och efter att ha avslutat varje steg av var och en kan utvecklaren skicka en ny commit från sin egen branch till chefen för att acceptera eller avvisa koden, och om den accepteras kommer den att slås samman

Flexbox - Vad är det? Hur fungerar det?

Foton hämtat från: ionos Redigerat: Okt. 7, 2022

- Flexbox är ett sätt att styra webbplatsens layout, teckensnitt och färgformatering, koordinering av kolumn- och radstorlekar.

- När vi skapar css-fil börjar vi sätta regler en efter en, t.ex. .rightcolumn {float: right;}.

- Om vi tar en titt på det högra "card" (Referenser & Kontakta mig) sattes den upp med flexbox, jag var tvungen att använda kommandon för att placera den på rätt plats och lämna en marginal från alla sidor. "card { background-color: white; padding: 20px; margin-top: 20px;".

DOM (Document Object Model) - Vad är det? Hur manipulerar man DOM?

Foton hämtat: freecodecamp Redigerat: Okt. 7, 2022

- Nu pratar vi om JavaScript! DOM (Document Object Model) är en plattform för att göra webbplatsen mer dynamisk, "The W3C Document Object Model (DOM) is a platform and language-neutral interface that allows programs and scripts to dynamically access and update the content, structure, and style of a document." (W3Schools, 2022).

- Var och en av dessa noder har sina egna egenskaper och metoder som kan manipuleras med JavaScript

- När vi besöker en webbsida via webbläsare som chrome, och vi klickar på högerklicka och väljer inspektera, kan vi se strukturen på sidan (innehållet) på detta sätt som kallas träd som är som mindmap. Vi kan se header, body, footer på sidan med koden.

Fin kod - Vad menas med det begreppet? Varför är det viktigt?

Foton hämtat från: geeksforgeeks Redigerat: Okt. 7, 2022

- Den fin kod är ett sätt att koda på som kan vara förståeligt av den andra programmeraren.

- Det är viktigt eftersom det sparar tiden när det är lättläslig.

- Det kan vara genom att separera koder från varandra, kommentera några koder för att förklara idén av det.

Ramverk - Inom webbutveckling använder man begreppet ramverk, vad är det? Beskriv vad det är och berätta lite mer om två valfria.

Foton hämtat från: konstantinfo Redigerat: Okt. 7, 2022

- Ramverk är "En kod som någon annan har kodat och som jag kan återanvända vilket gör att jag vinner mycket tid." (JU's canvas).

- T.ex. Bootstrap och React.

- React: Genom att installera node och lägga till React till Visual Studio kan vi importera koder från React-library "import React from "react"; ".

- Bootstrap: det finns många versioner av det, den senaste är Bootstrap 5, man kan länka HTML filen till Bootstrap library (https://getbootstrap.com/docs/5.2/getting-started/introduction/) och inkludera JS kod t.ex.
